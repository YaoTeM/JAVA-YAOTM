  ### 项目经历

  #### **Ragent - 企业级智能体平台** | `SSE` `Milvus` `RAG` `Redisson` `CompletableFuture` `TTL` `Tika`

  - **项目描述:** Ragent 是一个企业级 RAG 智能体平台，围绕智能文档检索与问答场景，提供多路检索引擎、意图识别、问题重写、会话记忆、MCP 工具调用等核心能力。系统采用多模型路由与熔断降级机制，支持高并发场景下的稳定服务。通过分布式队列限流和全链路追踪，保障系统在并发下的可用性与可观测性。
  - **核心工作:**
    - **多路检索**：意图定向 + 全局向量双路召回，CompletableFuture 并行执行，去重与重排序后处理，兼顾召回率与精准度。

    - **限流与高可用**：Redis（ZSET + Lua + Pub/Sub）实现分布式排队限流与公平排队；三态熔断 + 优先级降级 + 流式首包探测，多模型自动切换，用户无感知。

    - **上下文与追踪**：TransmittableThreadLocal 透传用户/Trace 上下文，8 个专用线程池 + TtlExecutors 包装，保证异步场景下全链路追踪完整。

    - **对话与记忆**：LLM 问题重写与拆分、多轮上下文补全；滑动窗口 + 超阈值摘要持久化 MySQL + TTL，控制 Token 成本。

    - **入库与可观测**：节点编排的文档入库流水线（拉取→解析→分块→增强→向量化），可配置、可追踪；AOP 全链路 Trace，关键节点记录耗时与异常。

    - **扩展能力**：MCP 工具集成、树形意图分类（KB/MCP/SYSTEM）、双维度幂等（framework 层），检索通道与 MCP 工具可插拔，零配置注册。

  #### **SaaS 短链接管理系统** |  `Spring Boot` `Spring Cloud Gateway` `MySQL` `Redis` `ShardingSphere` `Sentinel`

  - **项目描述:** 面向企业和个人用户的高并发 SaaS 短链接管理平台，提供短链生成、高性能跳转、分组管理以及多维度数据监控（地区、设备、网络等）分析功能。
  - **工作内容:**
    - **分库分表**： 引入 ShardingSphere，按 Hash 分片对短链及路由表分表，提升读写性能。
    - **高并发跳转与防穿透**： 短链请求采用 302 重定向；用 Redis 缓存长链，并基于 布隆过滤器 拦截非法短链，避免缓存穿透。
    - **防击穿与异步统计**： 缓存重建时使用 Redis 分布式锁 防止热点 key 击穿；用 Redis Stream 异步落库访问统计（UV/PV、IP、设备等），降低跳转 RT、提升吞吐，并保证消费幂等。
    - **网关与限流**： 基于 Spring Cloud Gateway 统一路由，配合 Sentinel 动态限流，保障流量峰值下系统可用性。
