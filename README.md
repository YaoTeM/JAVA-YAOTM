# Hot-100
力扣刷题-热题100

在软件工程中，确实有一套非常权威且被各大开源项目和企业广泛采用的固定模版——**约定式提交（Conventional Commits）**，它最早由 Angular 团队推行。

如果你在个人的项目（特别是写在简历上的项目）中严格使用这套规范，会让查看你 GitHub 提交记录的面试官立刻感受到你具备良好的工程化素养和团队协作意识。

### 约定式提交的标准格式

整体的提交消息由三个部分组成：**Header（标题行）**、**Body（正文）** 和 **Footer（脚注）**。其中只有 Header 是必填的。

我们可以用下面这个字符画来直观拆解它的结构：

```text
  [ 提交类型 ] ( [ 影响范围 ] ) : [ 简短描述 ]    <--- Header (必填，通常限制在 50-72 个字符)
       |             |                |
       |             |                +---> 动词开头，简明扼要说明“做了什么”
       |             +---> (可选) 修改的特定模块，比如 controller, config, user
       +---> (必填) 标记本次提交的性质，如 feat, fix

  [ 空行 ]

  [ 详细描述 (Body) ]                        <--- Body (可选)
       |
       +---> 补充说明：为什么要做这个改动？解决了什么具体问题？
             (适合逻辑较复杂，需要向未来的代码维护者解释意图时填写)

  [ 空行 ]

  [ 脚注 (Footer) ]                          <--- Footer (可选)
       |
       +---> 通常用于关闭项目管理工具中的任务，例如：Fixes #123, Closes #456

```

### 核心词汇：7 大常用提交类型（Type）

在日常开发中，只要掌握这几个最常用的前缀，就完全足够应对绝大多数场景了：

* **`feat`**: 新增功能（Feature）。
* *示例*：`feat(user): 添加用户注册与 JWT 登录接口`


* **`fix`**: 修复 Bug。
* *示例*：`fix(order): 修复并发扣减库存时出现的超卖问题`


* **`docs`**: 仅仅修改了文档（如 `README.md`、接口 API 文档）。
* *示例*：`docs: 更新项目本地环境的启动说明`


* **`style`**: 代码格式调整（不影响业务逻辑，比如去除多余空格、格式化代码、处理 Checkstyle 警告）。
* *示例*：`style: 统一 Controller 层的代码缩进`


* **`refactor`**: 代码重构（既不是新增功能，也不是修复 bug 的代码变动，比如提取公共方法、优化设计模式）。
* *示例*：`refactor: 将密码加密逻辑抽取为单独的工具类`


* **`test`**: 增加或修改测试用例（不涉及生产代码的改动）。
* *示例*：`test(service): 补充 UserService 的单元测试覆盖率`


* **`chore`**: 构建过程、配置文件或辅助工具的变动（如更新 Maven/Gradle 依赖）。
* *示例*：`chore: 升级 Spring Boot 版本至 3.2.0`



### 优秀实践与避坑指南

1. **原子性提交**：千万不要把写了三天的代码积攒成一个巨大的提交，然后写一个 `feat: 完成了好多功能`。尽量做到“一个功能/一个 Bug 对应一次提交”。
2. **用词习惯**：简短描述部分（Subject）最好以**动词**开头（如“添加”、“修复”、“优化”、“重构”），并且不要在句末加句号。
3. **注意冒号后面的空格**：规范要求在类型/范围和描述之间，冒号后面**必须有一个空格**（即 `feat: xxx`，而不是 `feat:xxx`）。

当你习惯了这种格式后，可以通过一些自动化工具（比如基于 `git log` 的脚本）一键生成非常漂亮的 `CHANGELOG.md`（版本更新日志）。

在实际开发中，如果我们在本地频繁保存代码，可能会产生很多杂乱的临时提交（比如 `fix: typo`，`fix: 又改了一个小bug`）。你需要我们顺着这个话题，过一遍如何使用 `git rebase -i` 命令，在把代码推送到远程仓库前，将这些琐碎的提交**“合并（Squash）”**成一个干净漂亮的提交记录吗？这也是高级开发中非常实用的一招。
